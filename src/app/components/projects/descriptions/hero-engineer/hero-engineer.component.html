Project Summary

HeroEngineer.com was an experimental learning environment for USC's "Advanced Writing For Engineers" course. It provided an interactive experience based around the concept of heroes in engineering, using a quest system with quizzes, assignments, avatars, and team interactions to explore what it means to be ethical and heroic as an engineer.

Personal Summary

  • Improved my soft skills and my ability to gather requirements and improve the project based on feedback -- with 100+ emails between the client and myself, and dozens of iterations that addressed professor and student feedback
  • Gained exposure to devops and sysadmin tools and processes (cron job for backups, Nginx for routing web traffic, tmux for organization/sessions, and SSL certificates on a remote Linux machine for security)
  • Gained technical support experience by owning the whole stack and being responsible for resolving bugs and other confusions for 50+ students during 3 semesters
  • Furthered my software engineering and creative abilities by designing the UI/UX and developing the site from start to finish (using MongoDB, React redux, Java Spring, and Gimp)

Key Features

For students:
  • Customizable accounts and avatars (using SVGs) with an interactive process for online messaging and team formation for group projects
  • Live, in-class assignments and quest system of unlockable quizzes that allow for rapid feedback on correct/incorrect answers
For professors:
  • Class sections and rosters -- only allow verified students with @usc.edu email addresses to sign up, and divide them into class sections
  • Automatic quiz grading, as well as manual grading with an option to provide feedback for subjective assignments (for example, questions that require paragraph answers rather than True/False answers)
  • Extensive settings for creating quests and quizzes and for customizing the order in which they must be taken
